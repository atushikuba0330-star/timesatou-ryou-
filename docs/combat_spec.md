# 戦闘システム仕様メモ

このドキュメントは、実装済みのコードを見ながら「本来どう動くべきか」を言語化して整理していくためのメモです。
コードとズレている部分が見つかったら、ここに正としての仕様を書き、そのあとコードを直す進め方をします。

## 1. ターン進行とタイミング

### 全体のループ
- `mana_manager.gd` が3秒ごとにループしており、これが戦闘全体の「心臓の鼓動」にあたる。
- 1周期(3秒)ごとに起きること:
  1. 自分/敵のマナが1回復する
  2. 全スロットの `progress_turn()` が呼ばれる(詠唱の進行・状態遷移)
  3. `battle_manager.resolve_turn()` が呼ばれ、`READY_TO_BATTLE` 状態のスロットがあれば戦闘解決される

### スロットの状態遷移
`EMPTY → CHANTING → COMPLETE → READY_TO_BATTLE`

- `CHANTING`: カードを配置してから、詠唱数(`actual_cast_time`)に達するまでの間。周期ごとに `chant_progress` が1ずつ増え、それに合わせて魔法陣(`magic_circles`)が1つずつ表示されていく。
- `COMPLETE`: `chant_progress` が `actual_cast_time` に到達した周期に入る状態。**この瞬間、魔法陣が全部揃った状態になる。**
- `READY_TO_BATTLE`: `COMPLETE` になった**次の周期**で遷移する。遷移したのと同じ周期内で `resolve_turn()` によって戦闘が解決される。

### 【仕様確定】COMPLETE〜READY_TO_BATTLEの1周期(3秒)の意味
これは不具合ではなく意図した仕様。この1周期(3秒)の間に、
- 完成した魔法陣を見せる時間
- 攻撃(エフェクト/projectile)が飛んでいく時間

を両方確保するためのタメとして使う。つまりエフェクトの発射タイミングは「戦闘解決の瞬間」ではなく「詠唱完了(`COMPLETE`に入った瞬間)」にする。3秒かけて飛んでいき、次の周期の戦闘解決で着弾・ダメージ確定、という流れに合わせる。

### 同時発動(相殺)の条件
`resolve_pair`(1対1の潰し合い)になるのは、**自分と相手(`enemy_slot`)の詠唱が同じ周期で完了し、同じ周期で`READY_TO_BATTLE`に揃った場合のみ**。
1周期でもズレると、片方が先に `resolve_vs_chanting`(詠唱中/シールド/空きスロット相手の処理)として単独で解決されてしまう。
→ エフェクト演出でも、「自分と相手が同じ周期で`COMPLETE`になったかどうか」を見て、
  - 揃っていれば: 双方から中間地点へ2本飛ばして衝突させる
  - 揃っていなければ: 自分から相手のスロットへ1本飛ばす
を出し分ける必要がある。

## 3. 戦闘解決ルール

### 1対1(`resolve_pair`)
- 威力の差分だけダメージが入り、負けた側のカードが壊れる。
- 同威力なら相打ちで両方壊れる。
- 【要修正】勝敗判定は本来 `is_player` を見て「勝った側がどちらのHPを削るか」を決めるべきだが、現状は威力差分の正負だけで `damage_enemy`/`damage_player` を固定で呼び分けており、`PlayerSlot` が `EnemySlot` よりシーンツリーで先にある偶然の並び順に依存している。→ 明示的に `is_player` で判定するよう修正する。
- 【要修正】相打ち(同威力)の場合、現状は両方のアビリティが発動しない。**本来は相打ちでも両方のアビリティを発動させる。**

### 片側のみ(`resolve_vs_chanting`)
- シールドが無い相手: 相手の詠唱中カードの威力を引いた分がダメージになり、相手のカードは中断されて壊れる。
- シールドがある相手: 下記「シールド仕様」を参照。

### シールド仕様(確定)
- シールドのアビリティを発動すると、**発動したカード自身のスロット**にシールドが付与される(`shield_value`)。
- ダメージを受けるとき、シールド値×10%だけダメージをカットする。カット後のダメージは**最低0(マイナスにはならない=回復はしない)**。
- シールドが付与されているスロットに置かれたカードは、攻撃を受けても**壊れない**。このとき参照するのは攻撃側の威力のみで、シールド側のカードの威力は参照しない。
- シールドは一度攻撃を受けると消費されて0に戻る(使い切り)。
- 【方針転換】「カードが無くなったらシールドも消える」は一度実装したが撤回した。理由: シールドをかけたカード自身も、役目を終えたら普通に`destroy_card()`で消える。もしこのタイミングでシールドも消してしまうと、シールドをかけた瞬間に自分で消してしまうことになり、アビリティとして機能しなくなる。→ シールドは「攻撃を受けて消費されるまで」スロットに残り続ける今の挙動のままにする(カードが無い空きスロットでも盾は生きている状態を許容する)。
- 【要修正】現状、同時発動(自分と相手が同じ周期で`READY_TO_BATTLE`)で相手にシールドがある場合、シールド側スロットは「守るだけ」で終わらず、同じ周期内でもう一度「攻撃側」としても独立処理されてしまい、結果的に2回分のダメージが出てしまう。
  → 修正方針: シールドで攻撃を受けたスロットは、その周期内では攻撃側として処理しない(`processed`に加えるが解決はしない)。シールドで攻撃を受けたカードは壊れずその場に残るので、次の周期になって初めて(シールド無しの状態で)通常通り攻撃・潰し合いができるようになる。

## 7. 見つかった不具合・修正履歴
- [修正済み] カードノードの `_gui_input` が、手札のカードと配置済みのカードを区別していなかったため、相手スロットのカードを奪えたり、自分の詠唱中カードを詠唱進捗ごと複製できてしまっていた。`get_parent().is_in_group("slots")` で配置済みかどうかを判定し、配置済みならドラッグ開始しないように修正。
- [修正済み] `resolve_pair` の勝敗判定がシーンツリーの並び順に依存していた。`a.is_player` / `b.is_player` を明示的に見て `damage_enemy`/`damage_player` を呼び分けるように修正。
- [修正済み] 相打ち(同威力)のとき、両方のアビリティが発動しないようになっていた。`finish_card_with_ability` に `count_as_win`(初期値true)引数を追加し、相打ちのときは `false` を渡してアビリティは発動させつつ必殺技カウントには含めないように修正。
- [修正済み] 同時発動で相手にシールドがある場合、シールド側スロットが「守る」処理と「攻撃する」処理の両方で独立に解決され、二重にダメージが出ていた。`resolve_turn()` でシールド側の相手も `processed` に加えて、その周期内では攻撃側として再処理されないように修正。カードは壊れず残るので、次の周期(シールド無し)で改めて攻撃・潰し合いができる。
- [修正済み] シールドのダメージカットで、シールド値が高いとカット後のダメージがマイナス(=回復)になり得た。`max(diff - cut, 0)` で最低0に固定。
- [実装→撤回] カードが破壊/移動でスロットからいなくなったらシールド値も消すよう `destroy_card()` と `_move_to_random_slot()` に `shield_value = 0` を追加したが、シールドをかけたカード自身の`destroy_card()`呼び出しで、付与した直後のシールドを消してしまう不具合が発生したため撤回。空きスロットに盾が残り続ける挙動はそのまま許容する。
